# <a name="site-and-page-navigation-in-sharepoint-sites"></a>SharePoint 사이트의 사이트 및 페이지 탐색

<p>사이트 및 페이지 탐색에 대한 기본적인 원리와 좋은 사례를 클래식 및 최신 SharePoint 아키텍처에 동일하게 적용할 수 있습니다. 그러나 탐색을 <em>구현</em>하기 위한 옵션은 사이트와 인트라넷의 프레임워크에 따라 다릅니다. 예를 들어 최신 &ldquo;환경&rdquo; 에서는 클래식 SharePoint 사이트 계층 구조에서 사용할 수 있는 상속 된 탐색 환경, 즉 하위 사이트가 있는 사이트는 사용할 수 없지만 <a href="https://support.office.com/article/fe26ae84-14b7-45b6-a6d1-948b3966427f" data-linktype="external">허브 사이트</a> 는 사이트 간 효과적인 방법을 제공 합니다. 이전에는 클래식 SharePoint의 관리 탐색 및 사이트 계층 구조에서 사용할 수 있는 탐색 기능을 제공 합니다. 탐색 옵션에 대 한 자세한 내용을 보려면 아래 링크를 방문 하세요.</p> <ul> <li><a href="https://docs.microsoft.com/en-us/sharepoint/plan-navigation-modern-experience">최신 SharePoint 환경에 대 한 탐색 계획</a></li> <li><a href="https://support.office.com/en-us/article/customize-the-navigation-on-your-sharepoint-site-3cd61ae7-a9ed-4e1e-bf6d-4655f0bf25ca">SharePoint 사이트의 탐색 사용자 지정</a></li> <li><a href="https://docs.microsoft.com/en-us/office365/enterprise/navigation-options-for-sharepoint-online">SharePoint Online에 대 한 탐색 옵션</a></li> <li><a href="https://techcommunity.microsoft.com/t5/Microsoft-SharePoint-Blog/Organize-your-SharePoint-sites-with-megamenu-navigation-and-new/ba-p/328068">Megamenu 탐색을 사용 하 여 SharePoint 사이트 구성</a></li> <li><a href="https://docs.microsoft.com/en-us/sharepoint/dev/general-development/managed-metadata-and-navigation-in-sharepoint">SharePoint의 관리 되는 메타 데이터 및 탐색</a></li> </ul>
