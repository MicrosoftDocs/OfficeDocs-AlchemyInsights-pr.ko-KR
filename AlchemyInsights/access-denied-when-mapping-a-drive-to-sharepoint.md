# <a name="fix-problems-with-sharepoint-libraries-mapped-to-network-drives"></a>네트워크 드라이브에 매핑되는 SharePoint 라이브러리와 문제 수정

매핑된 네트워크 드라이브를 검색할 때 다음 메시지 중 하나가 나타날 수 있습니다.
  
- **\\경로 액세스할 수 없는 경우 이 네트워크 리소스를 사용할 수 있는 권한이 없을 수 있습니다. 액세스 권한이 있는지 확인 하려면이 서버의 관리자에 게 문의 합니다.**
    
- **액세스가 거부 되었습니다. 이 위치에 파일을 열기 전에 하면 해야 먼저 신뢰할 수 있는 사이트 목록에 웹 사이트를 추가, 웹 사이트로 이동 하 고 자동으로 로그인 하는 옵션을 선택 합니다.**
    
[문제해결 도움말 매핑된 네트워크 드라이브를 사용](https://support.office.com/article/ef399c67-4578-4c3a-adbe-0b489084eabe.aspx)합니다.
  
임시 및 Internet Explorer 에서만에서 지원 되는 네트워크 드라이브로 라이브러리 매핑 (영문)입니다. 대신, [새 OneDrive 동기화 클라이언트와 SharePoint 파일을 동기화](https://support.office.com/article/6de9ede8-5b6e-4503-80b2-6190f3354a88.aspx) [주문형 파일을](https://support.office.com/article/0e6860d3-d9f3-4971-b321-7092438fb38e.aspx)포함 합니다. 로컬 저장 공간을 사용 하지 않고 OneDrive에서 모든 파일에 액세스 합니다.
